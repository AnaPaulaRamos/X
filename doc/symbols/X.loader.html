Class Detail
X.loader()
This object loads external files in an asynchronous fashion. In addition, the loading process is monitored and summarized to a total progress value.
Field Detail
<static> X.loader.extensions
Supported data types by extension.
Method Detail
addProgress(value)
Add a value to the overall progress value. The given value is normalized according to the number of current jobs. After adding the value, a X.event.ProgressEvent is fired to inform the progress bar.
Parameters:
{!number} value
The value to add.
{!Array} checkFileFormat(container)
Check if the given container is configured with a valid file format.
Parameters:
{!X.base} container
A container which has an X.file() attached.
Throws:
{Error}
An error, if the configured file format is not supported.
Returns:
{!Array} An array holding the following information [filepath, extension in uppercase without the '.', the associated but uninstantiated parser for this file format, additional (optional) flags which are passed to the parser, the response type]
complete(event)
The complete callback which gets executed after successful parsing of a data stream. A X.event.ModifiedEvent is fired to inform the X.renderer that the loading was completed.
Parameters:
{!X.event.ModifiedEvent} event
The modified event holding container and object.
{boolean} completed()
Check if all loading is completed.
Returns:
{boolean} TRUE if all loading is completed, FALSE else wise.
failed(request, container, object)
The failure callback which gets executed if anything goes wrong during downloading. It always throws an error.
Parameters:
{!XMLHttpRequest} request
The original XHR.
{!X.base} container
The container which has a X.file() attached.
{!X.object} object
The X.object which is the parent of the container or equals it.
Throws:
{Error}
An error, stating that something went wrong.
load(container, object)
Download a file associated to a container via Ajax. Callbacks are executed depending on the success of the request.
Parameters:
{!X.base} container
The container which has a X.file() attached.
{!X.object} object
The X.object which is the parent of the container or equals it.
Throws:
{Error}
An error, if the given objects were invalid.
parse(request, container, object)
Trigger parsing of a data stream. This is the callback for successful downloading. A single-shot listener is configured for a X.event.ModifiedEvent which then triggers the complete callback.
Parameters:
{!XMLHttpRequest} request
The original XHR.
{!X.base} container
The container which has a X.file() attached.
{!X.object} object
The X.object which is the parent of the container or equals it.
