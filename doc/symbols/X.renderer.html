The callback for X.event.events.MODIFIED events which re-configures the object for rendering. This does not trigger re-rendering.
Parameters:

The progress event holding the total progress value.

{!X.event.ScrollEvent} event
The scroll event indicating the scrolling direction.

{Error}
An error if the given event is invalid.
onShowtime()
Overload this function to execute code after all initial loading (files, textures..) has completed and just before the first real rendering call.
printScene()
Print the full hierarchy tree of objects.
render()
(Re-)render all associated displayable objects of this renderer. This method clears the viewport and re-draws everything by looping through the tree of objects. The current camera is used to setup the world space.
render_(picking, invoked)
Internal function to perform the actual rendering by looping through all associated X.objects.
Parameters:
{boolean} picking
If TRUE, perform picking - if FALSE render to the canvas viewport.
{?boolean=} invoked
If TRUE, the render counts as invoked and f.e. statistics are generated.
Throws:
{Error}
If anything goes wrong.
resetViewAndRender()
Resets the view according to the global bounding box of all associated objects, the configured camera position as well as its focus _and_ triggers re-rendering.
showProgressBar_()
Shows the loading progress bar by modifying the DOM tree.
update_(object)
Configure a displayable object within this renderer. The object can be a newly created one or an existing one. A X.renderer.render() call has to be initiated to display the object.
Parameters:
{!X.object} object
The displayable object to setup within this renderer.
Throws:
{Error}
An exception if something goes wrong.
